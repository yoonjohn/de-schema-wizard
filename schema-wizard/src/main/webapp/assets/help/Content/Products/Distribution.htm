Distribution

The DigitalEdge Schema Wizard is packaged in three docker containers: an H2 database container, a reverse geocode container, and a webapp container. H2 provides persistence for the ‘Catalog’ of schemas along with metrics for the data sample files used in schema generation. It also provides working space for the services. The reverse geocode container provides the services with a lookup capability for matching coordinates to country codes and utilizes a Mongo database with preloaded data. The webapp is a Jetty instance that serves the presentation layer and embeds the services layer. The presentation layer’s client-side is written in AngularJS and the server-side implements Jersey in a Spring environment running under Jetty. The services are written in Java.
