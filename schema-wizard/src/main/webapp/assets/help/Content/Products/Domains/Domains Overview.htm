Domains

One of the advanced features of the DigitalEdge suite of tools is the notion of 'domain'. Domains provide a set of interpretations that may be applied to fields in schemas. An interpretation provides semantic meaning for a field.

As an example, an initial analysis of values for a single field would provide syntactic meaning. Differentiation between string and numeric values is realized. With further analysis a numeric value may be recognized to fit the pattern of a longitude or a latitude. Statistical analysis of a sufficient number of samples could narrow the finding down to either a longitude or latitude. With the discovery of both a longitude and latitude in a single record an interpretation of 'coordinate' can be applied. Further interpretation could lead to determining a country, state, or region code. This progressive analysis provides an interpretation that has significant semantic meaning that far exceeds simple syntactic recognition.

This process of refining the interpretation of a field or group of fields can be guided when considering a specific domain. The semantic meaning of a field in the 'Health' domain could be given the interpretation of "procedure code". The same field in the 'Aerospace' domain may be recognized as a published "waypoint".

Each domain in the Schema Wizard has a collection of pluggable, interpretation descriptors. An interpretation engine loads the appropriate collection as set by the data engineer for each Schema Wizard session. Thus interpretations can be refined to an appropriate semantic meaning for that domain. The Schema Wizard provides a Domain Manager for maintaining these descriptors.
